AgentGate — Access Control for AI Agents

A concept prototype arguing that AI agents are a missing identity class in enterprise IAM — and sketching what a runtime governance layer would look like to fill that gap.

Type
Concept walkthrough / hackathon prototype
Role
Product strategy & concept design
Status
Concept — not a shipped product

The gap this is addressing

Most enterprise IAM frameworks are built around two identity classes: humans (governed by access request workflows, role assignments, and recertification) and services/applications (governed by service accounts, API keys, and machine credentials). Both classes have reasonably mature governance models.

AI agents don't fit cleanly into either. They act dynamically — a single agent might read from an inventory system, write to a finance platform, and call a vendor API in a single workflow — but they're typically granted access the way a service account would be: one set of credentials, broad scope, long-lived. The problem is that an agent's behavior is inherently variable in a way a conventional service's isn't.

Standing privilege plus dynamic behavior is exactly the combination that creates blast radius risk. AgentGate is the question: what if AI agents had the same governance model as humans — policy-based approval, temporary scoped access, and live revocation when behavior drifts?

The concept walkthrough

The walkthrough runs through eight slides: the gap, the wedge (why the familiar governance model transfers), the product concept (one runtime control point between agent and enterprise system), the differentiator (session-level revocation, not just gate-level approval), the MVP scope, and the value framing. It's deliberately positioned as a concept, not a spec — the goal is to name the governance gap clearly enough that the product conversation can start.

The argument in one paragraph

Enterprises already know how to govern human identities and service identities — they've built workflows, tooling, and audit infrastructure for both. AI agents need a third model that borrows from both: the approval-workflow discipline of human identity governance, and the credential-scoping discipline of service identity governance, plus one new capability that neither model requires today — live session revocation when behavior deviates from what was approved. That's the control plane that's currently missing, and it's missing because agents didn't exist as an identity class when the governance frameworks were designed.

Connection to real IAM experience

This concept isn't speculative in isolation — it's a direct extension of the NHI/NUA governance work in the experience timeline. Non-human identity governance (service accounts, API tokens, machine credentials) is already an unsolved problem at scale in most enterprises. AI agents make it more urgent because the consequences of a misconfigured agent credential are larger and faster than a misconfigured service account: agents can act across multiple systems autonomously before anyone notices something is wrong.

The PM question AgentGate is really asking is: how do you extend the access governance model you already have — rather than building a parallel system from scratch — to cover this new identity class?

Key elements of the concept

Control point
Runtime policy enforcement
One gate between the agent and the enterprise system — policy-based approve, escalate, or deny at the time of the access request, not at provisioning time.
Concept
Scoped access
Temporary, minimal grants
Approved access is time-bounded and scoped to the specific task — not a long-lived credential that covers everything the agent might ever need.
Concept
Live revocation
Session-level kill switch
If an active session shows behavioral drift — abnormal volume, lateral movement, scope creep — the session is revoked, not just flagged for review.
Key differentiator
Human in loop
Escalation for high-risk requests
High-risk requests (write access to finance systems, sensitive data) route to a human reviewer who can narrow scope or deny before access is granted.
